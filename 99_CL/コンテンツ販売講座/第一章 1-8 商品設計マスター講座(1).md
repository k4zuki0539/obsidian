## 2. 商品設計の2つの大前提

### 大前提1: 商品コンセプト通りに設計する

**鉄則:**

```
商品コンセプト通りに設計する
```

#### よくある間違い

**失敗パターン:**
- せっかく商品コンセプトを作ったのに、意識せずに商品を作る
- 商品設計と商品コンセプト設計を分離して考える

**→ これでは商品コンセプトを作った意味がない**

#### なぜコンセプト通りに作るべきか?

**重要な原則:**

```
商品コンセプトの良し悪し = 商品の売れ行き
```

| 商品コンセプト | 結果 |
|--------------|------|
| 良い商品コンセプト | 売れる商品 |
| ダメな商品コンセプト | 売れない商品 |

**→ 商品コンセプト設計の方が商品設計よりも重要**

### 大前提2: 必ずゴールを達成させる

**商品の目的:**

```
顧客のゴールを達成できるコンテンツを作る
```

#### ゴールの例

| ジャンル | ゴール |
|---------|--------|
| 美肌 | 3ヶ月でニキビをなくす |
| SNS収益化 | 6ヶ月で月収50万円稼げるようにする |
| 転職 | 3ヶ月で大手企業に転職させる |
| 動画編集 | 1ヶ月で動画編集案件を獲得させる |
| Webデザイン | 2ヶ月でWebデザイン案件を獲得させる |

**重要:**
- ゴールを必ず達成させることができるように商品を作る
- ゴールが達成できない商品 = 悪い商品(詐欺商品)

**→ これは商品設計の最重要命題**

## 3. 商品設計でやること

### 商品設計は難しくない

**商品設計でやることはシンプル:**

```
商品コンセプトで決めたゴールを、最短で達成させるための具体的な流れを作り込む
```

**→ これだけ**

### よくある誤解

**誤解:** 商品設計は難しい

**真実:**
- 商品を作ること自体は難しくない
- 難しいのは「作り切ること」

**例:**
- スライドを作り込む
- 動画を撮影する
- コンテンツを完成させる
- → これらの作業が大変

**→ しかし、商品を作ること自体は簡単**

### 最短でゴールを達成させる

**重要なマインド:**

期間設定より早く達成させる意識

**例:**
- 6ヶ月で転職というゴールなら、3ヶ月で転職できるように成長させる
- → 最短で達成させるという意識

### ゼロイチ段階の商品設計は超簡単

**ゼロイチの定義:**
- まだゴールを達成していない
- 月収10万円程度の収益段階

**ゼロイチ段階の鉄則:**

```
ゼロイチ(月収10万円以下) = 必ずマンツーマンにする
```

**理由:**
- 講座を作り込む必要がない
- 直接教えられる
- 柔軟に対応できる

**講座形式に移行するタイミング:**
- 月収100万円以上を安定させてから
- マンツーマンコンサルで十分な実績を積んでから

## 4. ゼロイチはキュレーションでOK

### キュレーションとは?

**定義:**

```
いろいろな情報を収集して、必要な部分のみをギュッとまとめる作業
```

**情報源:**
- 書籍
- SNS上の有用な情報
- 他の講座
- 自分の経験

**→ これらをまとめて商品にする**

### なぜキュレーションでOKなのか?

#### 理由1: 情報はすでに存在している

**現代の状況:**
- 情報はあふれている
- 必要な情報はすでに存在している
- 問題は「情報にたどり着けない人が多い」

#### 理由2: コンテンツ販売の本質

**コンテンツ販売の本質:**

```
情報過多の時代に、キュレーションで正しいノウハウだけをギュッと凝縮して届けること
```

**提供する価値:**
- 情報収集のコストを削減
- 最短で理想の未来を手に入れさせる
- サポートで寄り添う

### 実例: このコースもキュレーション

**コンパスの商品も:**
- キュレーションで作られている
- 1,500万円の自己投資から得た情報をまとめたもの
- 自分の知識・経験・体系化スキルを加えている

**→ どの商品も基本的にキュレーション**

### ゼロイチの商品設計はこれだけ

**ゼロイチの商品設計:**

1. キュレーションで情報を集める
2. マンツーマンコンサルをする
3. 結果を出させる

**→ これだけでOK(商品設計はほぼ不要)**

## 6. ステップ1: 目次と提供内容の洗い出し

### ステップ1の目的

**何をするか:**

```
大まかな商品の全体像を設計する
```

**具体的には:**
- 商品コンセプトで決めたゴールを最短で達成するための具体的な流れを考える

**ゼロイチの人:**
- **ここだけでOK**
- マンツーマンコンサルなら、これ以降は不要

### 3つの目次を作る

#### 大目次・中目次・小目次

**構造:**

```
大目次
 ├─ 中目次1
 │   ├─ 小目次1-1
 │   ├─ 小目次1-2
 │   └─ 小目次1-3
 ├─ 中目次2
 │   ├─ 小目次2-1
 │   └─ 小目次2-2
 └─ 中目次3
```

### 具体例1: SNSマーケティングマスター講座

**大目次:**
「SNSマーケティングマスター講座」

**中目次:**
- 第0章: イントロダクション
- 第1章: ビジネス基礎
- 第2章: ジャンル選定
- 第3章: 自社分析
- ...
- 第10章: 実践

**小目次(例: 第1章):**
- 1-1: ビジネスマインドセット
- 1-2: SNSビジネスの全体像
- 1-3: 収益化の仕組み

**目的:**
- 6ヶ月で7桁を稼がせる
- → このゴールから逆算して作成

### 具体例2: キャンバデザインマスター講座

**目的:**
- キャンバデザインをマスターさせる

**構成:**
- ゴールから逆算して講座を設計
- 最短でキャンバデザインをマスターできる流れ

### 具体例3: コンテンツゼロイチマスターコース

**大目次:**
「コンテンツゼロイチマスターコース」

**中目次:**
- 第1章: ビジネス準備期マスター
- 第2章: 商品構築期マスター
- 第3章: 集客期マスター

**小目次(例: 第1章):**
- ビジネススキルマスター
- マインドセットマスター
- SNS運用基礎マスター
- ...
- 個別コンサルティングマスター講座

**目的:**
- 最短3〜4ヶ月でゼロイチを達成
- 6ヶ月で月収10万円は現実的
- ゼロイチを完全テンプレート化してクリアさせる

### 洗い出しのポイント

**重要:**
- ゴールから逆算する
- 最短で達成するための流れを考える
- 大まかでOK(詳細は後で詰める)

## 7. ステップ2: コンテンツ内容の設計

### ステップ2の目的

**何をするか:**

```
小目次の中身を具体的に作る
```

**対象者:**
- 講座形式で提供する人
- マンツーマンコンサルの人は不要

### コンテンツ内容設計の4ステップ

| ステップ | 内容 |
|---------|------|
| 2-1 | ラフ設計(情報収集) |
| 2-2 | 体系化 |
| 2-3 | 清書 |
| 2-4 | コンテンツ化 |

### ステップ2-1: ラフ設計(情報収集)

#### 情報を大量に集める

**情報源:**
- 書籍
- Google検索
- SNSの無料情報
- 自分が持っている知識

**やること:**
1. 小目次のテーマについて、情報を大量に集める
2. マインドマップにとにかく書き出す
3. 順不同でOK

**例: 大ジャンル選定の講義を作る場合**
- 大ジャンル選定について調べまくる
- 7〜10人分の情報を集める
- Google検索で5ページ目まで見る
- すべてマインドマップにまとめる

#### 知識がある人は自分の知識で作る

**知識がある人:**
- キュレーションしなくてOK
- 自分の知識だけで作った方が早い
- → これがオリジナルの商品

**知識がない人:**
- キュレーションで情報を集める必要がある

### ステップ2-2: 体系化

#### なぜ体系化が必要か?

**問題:**
- ラフのままだとバラバラ
- バラバラのまま講義にすると、めちゃくちゃ分かりにくい

**解決:**
- 順序立てて説明できるように並び替える

#### 体系化の構造

**推奨する構造:**

1. **はじめに(導入)**
   - 目的・ゴール
   - この講座のメリット
   - 前提条件のすり合わせ

2. **全体像の把握(結論)**
   - 結論を先に言う
   - 全体像を示す

3. **本編(解説)**
   - 1つずつ解説していく
   - ステップバイステップ

4. **補足・まとめ**
   - 最後に補足があれば追加

**例: この講座の構造**
- はじめに → 商品設計の前提
- 全体像 → 5つのステップでOK
- 本編 → 1つずつ解説
- まとめ → 重要ポイントの再確認

**→ わかりやすい講座になる**

#### 体系化の方法

**PREP法や起承転結を活用:**
- 結論を先に言う
- 理由を説明する
- 具体例を示す
- 再度結論を言う

### ステップ2-3: 清書

#### 清書とは?

**やること:**
- 体系化した内容を完成版にする
- 変な部分があれば修正する
- 足りない部分があれば追加する

**確認ポイント:**
- 誤字脱字がないか
- 言ってることが矛盾してないか
- わかりにくい表現がないか
- 足りない情報がないか

**→ 上から読んで確認する**

### ステップ2-4: コンテンツ化

#### コンテンツ化の3つの選択肢

| 形式 | 特徴 |
|------|------|
| スライド化 | 視覚的でわかりやすい |
| 記事化 | テキストで詳しく解説 |
| 動画化 | 動きと音声で伝わりやすい |

**マンツーマンコンサルの人:**
- **コンテンツ化は不要**
- ラフ設計までで十分

#### スライド化

**推奨ツール:**

| ツール | 特徴 |
|--------|------|
| Canva | 無料素材が豊富、テンプレートあり(推奨) |
| Googleスライド | 無料、共有が簡単 |
| Keynote | Macユーザー向け |

**→ Canvaが最もおすすめ**

#### 記事化

**推奨ツール:**

| ツール | 特徴 |
|--------|------|
| noteの限定公開 | 簡単、見やすい |
| GoogleドキュメントのPDF化 | 無料、共有が簡単 |
| 専用サイト(Kajabi, Teachable) | プロフェッショナル |

**→ noteまたはGoogleドキュメントがおすすめ**

#### 動画化

**画面収録ツール:**

| ツール | 特徴 |
|--------|------|
| QuickTime Player(Mac) | 標準搭載、無料 |
| Canva Video | Canvaスライドのみ録画可能 |
| Loom | 録画後すぐにリンク化、画面収録機能も優秀(推奨) |

**動画の格納先:**

| 格納先 | 特徴 | リスク |
|--------|------|--------|
| YouTube限定公開 | 無料 | ビジネス利用で消される可能性あり |
| Vimeo | 月1,000円程度 | 安全 |
| Googleドライブ | 無料 | 可能だが推奨度低い |

#### 最も楽な方法: Zoom録画

**推奨する方法:**

```
スライドだけ作る → Zoomで録画 → そのまま動画コンテンツ
```

**メリット:**
- ライブセミナーで話した内容がそのままコンテンツになる
- 事前に動画を撮影する必要がない
- めちゃくちゃ効率的

**やり方:**
1. スライドだけ作る
2. 1対1のZoomで講義する
3. 録画を取る
4. Zoomで書き出した動画をYouTubeやVimeoにアップ
5. 2人目以降は録画した動画を見てもらう

**→ これが一番楽で効率的**

**Zoom録画の注意点:**
- 自分の顔を画面オフにする
- 画面共有してからレコーディング
- → そうしないと右上に顔が映り続ける

### ステップ2-5: テンプレート作成

#### テンプレートの重要性

**なぜ必要か:**
- アクションに落とし込むことが最も大事
- アクションしないと結果が出ない
- テンプレートがあるとアクションしやすい

**テンプレートの例:**
- フレームワーク
- ワークシート
- チェックリスト

**マンツーマンコンサルの場合:**
- なくてもOK
- 直接教えて添削すればいい

#### テンプレート作成のコツ

**ステップバイステップにする:**

```
ステップ1 → ステップ2 → ステップ3 → ステップ4 → ステップ5 → 完成
```

**例:**
- 「この5ステップでOK」
- 各ステップを順番に進めるだけ

**→ わかりやすく、実践しやすいテンプレートになる**

## 8. ステップ3: ロードマップ・タスク化

### ステップ3の目的

**何をするか:**

```
顧客のカリキュラムを明確にする
```

**具体的には:**
- ゼロから100までのカリキュラムをロードマップ化
- 各章ごとのタスクを明確化

### ロードマップ・タスク化とは?

**ロードマップ:**
- 全体の流れを示すもの
- → 小目次がロードマップになる

**タスク:**
- 各講義ごとに何をすればいいかを明確化

**例:**

| 章 | 講義 | タスク |
|----|------|--------|
| 第1章 | 1-1 大ジャンル選定 | ・講義を視聴してアウトプット200文字以上<br>・大ジャンルを決定する |
| 第1章 | 1-2 ハッシュタグ戦略 | ・ハッシュタグを3パターン作る |
| 第2章 | 2-1 商品設計 | ・商品コンセプトを作り切る |

### なぜロードマップ・タスク化が必要か?

#### メリット

1. **自分もサポートを進めやすい**
   - タスクが明確
   - 何を教えればいいかわかる

2. **顧客も進めやすい**
   - 何をすればいいか明確
   - 迷わない

3. **サポートが最低限で済む**
   - 自分がタスク切りする必要がない
   - 効率的

4. **自分の時間も増える**
   - サポート時間が減る
   - 他のことに時間を使える

5. **成約率が上がる**
   - 不安要素がなくなる
   - ロードマップがあるから安心

**→ マンツーマンコンサルでもやった方がいい**

### マンツーマンコンサルの場合

**タスク管理の方法:**

```
自分がタスク管理をしてあげる
```

**例:**
「○○さんは、プロフィールが終わりましたね。次はハイライトを作ってください。ハイライトの作り方を解説していきます！」

**→ 絶対にタスク管理をする**

### やることが明確になる効果

**よくある不安:**
- 講座に入ったけど、何すればいいんだろう?
- マンツーマンコンサルを受けたけど、実際何するんだろう?

**ロードマップ・タスクがあると:**
「このロードマップ通りのタスクを行ってもらうことで、最短で結果を出させますよ」

**→ 成約率が上がる**

## 9. ステップ4: 提供スタイルの決定

**決めること:**

1. **保証の設計**

4. **サポート体制の設計**

**→ この4つを設計することが提供スタイルの決定**

**注:** 詳細は次の講座(第一章 1-8 商品設計マスター講座(2))で解説

## まとめ

### 商品設計の2つの大前提

1. **商品コンセプト通りに設計する**
   - コンセプトと設計を分離しない

### 商品設計でやること

```
商品コンセプトで決めたゴールを、最短で達成させるための具体的な流れを作り込む
```

### ステップ1: 目次と提供内容の洗い出し
