# オープンチャットの運用手順

## なぜオープンチャットを使うのか

### 1. ユーザーの視認率を最大化
- メンションオールで**強制通知**が可能
- オープンな場で一斉配信

### 2. 企画運営に必要な機能が揃っている
| 機能 | 用途 |
|------|------|
| アナウンス設定 | 重要な告知を固定 |
| ノートピン留め | 情報をまとめて保存 |
| メッセージ送信管理 | ユーザーの書き込みを制御 |
| 運営事務局アカウント追加 | 複数人で運営 |
| 投稿削除 | ネガティブコメント処理 |
| NGワード設定 | 不適切な書き込み防止 |

### 3. 参加しやすい
- LINEサービス内なので参加ハードルが低い
- 参加率が高くなりやすい

### 4. 体験型価値の施策に適している
- 1対1よりオープンな場の方が効果が高い
- 一体感の演出
- 帰属意識の教育

## オープンチャットで実施する7要素

```
1. 個別相談のスケジュールおさらい
2. 集客（SNS・ハウスリスト・広告 → OC一点集中）
3. サンクスページ動画の共有
4. OC内での動画配信（期待感・挨拶・事前課題）
5. 価値提供の配信（ライブセミナー）
6. 場のコントロール（追加コンテンツ等）
7. セールス・個別相談誘導・直売り
```

## オープンチャットの準備

### 初期設定
| 項目 | 内容 |
|------|------|
| トークルーム名 | 企画名に変更 |
| 背景 | 企画に合わせて変更 |
| アイコン | 企画用に設定 |
| 運営スタッフ | 事務局アカウント追加 |

### 最初にやること
1. ノート作成（「最初に必ずご確認ください」）
2. 企画の全体像を記載
3. 参加ルール
4. ライブ日程
5. 参加方法
6. よくある質問への回答

---

## 運用の流れ

### 集客期間

```
SNS・ハウスリスト
  ↓
誘導LP
  ↓
サンクスページ
  ↓
オープンチャット
```

#### やること
| アクション | 目的 |
|-----------|------|
| ノートを見るよう誘導 | 基本情報の周知 |
| 定期的なアナウンス | 参加を促す |
| 周知ノートの設定 | 質問を減らす |
| 事前課題の促し | 熱量維持 |

**ポイント：** ノートを見れば参加方法・問い合わせ方法がわかるようにする

---

### ライブ前

| 配信内容 |
|---------|
| ライブ前の興味付け |
| ライブで得られるメリット・ベネフィット |
| ライブ参加特典 |

---

### ライブ当日

| 配信内容 |
|---------|
| ライブの告知 |
| 内容・メリット・ベネフィット |
| ライブ参加リンク |
| ライブ参加特典 |

---

### ライブ終了後

| アクション | 内容 |
|-----------|------|
| 特典受け取りお知らせ | 受け取り方法 |
| アーカイブ共有 | URLを配信 |
| ノート更新 | 特典・アーカイブURL追加 |

---

### ライブ翌日（中日）

| アクション | 内容 |
|-----------|------|
| 課題を進めてもらう | アウトプット促進 |
| 課題フィードバック | 反応を返す |
| 未提出ユーザー退出処理 | スクリーニング（オプション） |
| 次回ライブ告知 | 期待感を持たせる |

---

### 繰り返し回数

| 型 | 繰り返し回数 |
|----|-------------|
| Day3型 | 2回 |
| Day5型 | 4回 |

※最終ライブ後の中日はなし

---

### ライブ全日程終了後

```
セールス追い → 申込期限終了まで継続
```

## イレギュラー対応

### 1. ネガティブコメントの処理

#### 基本方針
- 他のユーザーに目を触れさせない
- 運営権限で**投稿削除**

#### 対応パターン

| パターン | 対応 |
|---------|------|
| いわれのない批判・根拠のないネガティブ | すぐに削除 |
| 聞かれたくないけど正当な質問 | 誠意を持って返答 |

**ポイント：**
> ネガティブコメント1〜2件で成約率に影響することはない  
> 神経質になりすぎなくてOK

#### 逆効果になるケース
- パッと消してばかり → 信頼残高低下
- 誠意ある対応 → 逆に評価UP（Amazonカスタマーサポートの例）

---

### 2. NGワード設定

- 好ましくない質問に含まれるワードを登録
- 同じようなコメントを防止

---

### 3. 問い合わせ・質問の処理

#### 量が少ない場合
→ オープンチャット上でアナウンス対応

#### よくある質問例
- ライブに入れません
- 特典を受け取れません
- 動画を見れません
- 決済エラーが出ました
- 分割払いお願いしたい
- サポートはつきますか

#### 対応方法
| 方法 | 内容 |
|------|------|
| トピックごとにテキスト投稿 | 「ここで回答してます」と誘導 |
| 解決コンテンツ作成 | 動画・資料で共有 |

---

### 4. 1000人以上の場合

#### 問題点
- 意味のないやり取りが1000人に通知される
- 企画全体の期待値がマイナスに

#### 対策
- 問い合わせ用LINEを別で用意
- 個別質問用フォームを作成
- そちらに逃がす

## 運用期間の目安

| 企画 | 期間 |
|------|------|
| Day5型 | 募集〜セールス締め：最低2週間 |
| 長い場合 | 約1ヶ月 |

---

## 配信の注意点

### 予約配信ができない
- オープンチャットは**リアルタイム配信のみ**
- スケジュール通りに手動で投稿

### 対策
- 時間が取れない場合は誰かに依頼
- テンプレートを事前に用意

---

## ノートの活用

### 最初に用意するノート
```
重要1：最初に必ずご確認ください
- 企画の目的
- 参加ルール
- ライブ日程
- 参加方法
- 問い合わせ先
```

### ライブごとに追加
```
ワーク1：Day1の課題・特典受け取り
ワーク2：Day2の課題・特典受け取り
...
```

> ✅ ノートで基本情報をまとめて**質問を減らす**

> ✅ 1000人超えは**別で問い合わせ窓口**を用意
