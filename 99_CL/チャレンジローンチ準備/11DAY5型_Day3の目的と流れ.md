### Day3の特性
- 基本的にはDay2の構成と同じ

- Day4からセールスパートなので、ここまでの教育が重要

```
Day1・Day2の振り返り（厚めに）
```

## Day3特有のポイント

### 振り返りを厚めにやる

> ⚠️ Day1・Day2の前提を忘れられると、問題提起3の流れがユーザーに入らなくなる

**振り返る内容：**
- Day1の問題提起1と解決の流れ
- Day2の問題提起2と解決の流れ
- ユーザー自身が課題にたどり着いた経緯

**注意：** 間延びしない程度に、がっつり目に振り返る

---

### 問題提起3が最重要

> **問題提起3の解決策 = 商品**

ここで提示する課題が、商品の必要性を感じてもらうための核心部分

---

## 問題提起の流れ（AI活用の例）

### Day1（問題提起1）
```
課題：AIを実務に活用できない
理由：トレンド性のある情報ばかりで、実務に活かせる情報が見つからない
解決：情報のキャッチアップ方法を変える
```

### Day2（問題提起2）
```
課題：情報をキャッチアップできても、自分の業務に落とし込むのは別の技術
解決の方向性：業務ごとに最適なAI活用のパターンがある
アクション：ロードマップに業務を当てはめる
```

### Day3（問題提起2の解決 → 問題提起3）

#### 問題提起2の解決策（具体アクション）
```
・業務のジャンルごとに適切なAIの活用方法を解説
・フローチャートに沿ってジャンル分け
・それぞれのジャンルでのAI活用の仕方を教える
```

#### 問題提起3（新たな課題）
```
課題：活用の仕方は分かったけど...
  ・日々変化していく業務に対応するのが大変
  ・新しいAI活用方法がどんどんアップデートされる
  ・キャッチアップしながら最適な活用を試し続けるのはしんどい
```

#### 解決の方向性
```
→ この課題の解決策がDay4で紹介する「商品」
```

- Day4からセールスパートが入る
- Day3までの教育の質が成約率に影響

### 一体感の形成
- Day3からはリピーターがほとんど
- コミュニティとしての一体感が生まれる

| 項目 | Day2 | Day3 |
