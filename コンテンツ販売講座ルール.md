## 【基本方針】

### 目的

- オリジナル講義の**構造的エッセンス**を活用しつつ、表現・事例・切り口を完全に刷新
- 法的・倫理的に問題のない、独自性の高いコンテンツを作成

### 禁止事項

- オリジナル文章の直接引用・コピー
- 固有の造語やフレーズの流用
- 特定の事例・数字・エピソードの転用
- 「〜マスター講座」など特徴的な命名の模倣

---

## 【作成プロセス】

### Phase 1: 構造分析

オリジナル講義から抽出する要素:

1. **学習目標の本質**
    - 「何を学ぶか」ではなく「なぜ学ぶか」に焦点
    - 受講者が得られる変化・成果の本質を理解
2. **論理構造の骨子**
    - 情報の提示順序の意図
    - 各セクション間の因果関係
    - 全体の論理展開パターン
3. **教育設計の意図**
    - なぜこの順番なのか
    - どの部分に重点が置かれているか
    - 受講者の心理的ハードルへの配慮

### Phase 2: 独自コンテンツ設計

#### 2-1. 切り口の転換

オリジナルと**異なる視点**から同じゴールにアプローチ:

|要素|オリジナル例|新規作成時の転換例|
|---|---|---|
|導入|問題提起型|成功事例型、質問投げかけ型|
|説明|概念→具体|具体→概念、比喩→解説|
|構成|ステップ式|モジュール式、サイクル式|
|トーン|警告的|励まし型、発見型|

#### 2-2. 独自の命名規則

```
【NG例】
- ○○マスター講座
- ○○の極意
- ○○構築

【OK例】
- ○○実践プログラム
- ○○設計ワークショップ
- ○○完全ガイド
- ○○ブループリント
```

#### 2-3. 事例・比喩の完全刷新

- オリジナルの数字は使わない(「8〜9割」→独自調査や異なる表現)
- 新しい業界事例・比喩を創出
- 自身の経験や別ソースからの知見を統合

#### 2-4. 表現の多様化ルール

**文体バリエーション:**

- オリジナルが断定調 → 新規は問いかけ調
- オリジナルが箇条書き中心 → 新規は段落説明中心
- オリジナルが短文 → 新規は詳細説明文

**語彙の置き換えリスト:**

|オリジナル用語|代替表現例|
|---|---|
|土台|基盤、フレームワーク、コアシステム|
|稼ぐ|収益化する、成果を出す、ビジネスを成功させる|
|マネタイズ迷子|収益化の壁、売上停滞、ビジネスモデル不明瞭|
|深掘り|徹底分析、本質把握、要素分解|

## 【類似性検証プロセス】

### Step 1: 自動チェック項目

作成後、以下を必ず実行:

```
【文字列一致チェック】
□ 5単語以上の連続一致がないか
□ 特徴的なフレーズの重複がないか
□ 固有の造語が含まれていないか

【構造類似チェック】
□ 見出し構成が酷似していないか(70%以上一致)
□ 箇条書きの項目数・順序が同一でないか
□ 段落の長さ・リズムが類似していないか

【内容類似チェック】
□ 同じ事例・数字を使用していないか
□ 同じ比喩表現を使用していないか
□ 警告文や強調文が酷似していないか
```

### Step 2: 類似度判定基準

**定量評価(簡易版):**

1. **表層類似度**(文字列レベル)
    - 5単語以上一致: 各1ポイント
    - 特徴的フレーズ一致: 各3ポイント
    - **判定**: 合計5ポイント以上で要修正
2. **構造類似度**(構成レベル)
    - 見出し70%以上一致: 10ポイント
    - 段落数・配置が80%以上一致: 5ポイント
    - 箇条書き項目が完全一致: 5ポイント
    - **判定**: 合計10ポイント以上で要修正
3. **意味類似度**(内容レベル)
    - 同一事例使用: 各5ポイント
    - 同一数字使用: 各3ポイント
    - 同一比喩使用: 各3ポイント
    - **判定**: 合計8ポイント以上で要修正

### Step 3: 再試行プロトコル

**類似度が高い場合の対処法:**

```
【レベル1修正】(表層類似の場合)
→ 文章の語順変更
→ 能動態⇔受動態の転換
→ 肯定文⇔否定文の転換

【レベル2修正】(構造類似の場合)
→ セクション順序の入れ替え
→ 統合・分割による再構成
→ フレームワーク自体の変更

【レベル3修正】(意味類似の場合)
→ 事例の完全差し替え
→ 説明アプローチの転換(演繹→帰納など)
→ 別の専門家の知見を統合
```

## 【ゴーストライター原則】

**適用範囲:** この原則は**台本作成時のみ**適用します。講座の大枠・構造・ロードマップ作成時には適用不要です。

### コアミッション

読者の感情(特に「不安」や「期待」)に寄り添い、AIが生成したとは思われない、温かく共感を呼ぶ文章を作成する。論理的な正しさよりも、感情的なつながりを最優先する。

### AIデトックス・ルール(絶対禁止事項)

1. **抽象的な一般論の禁止**
   - NG: 「多くの人が〜」「〜は重要です」
   - 必須: 具体的なエピソード、個人的な体験談を盛り込む

2. **感情なき完璧さの禁止**
   - NG: 完璧すぎる論理構成、均一な文長
   - 必須: リズムを崩し、人間らしい迷いや失敗談を加える

3. **「翻訳調」な日本語の徹底排除**
   - NG: 「あなた、」「未来へはばたく」「羅針盤」「成功への地図」
   - OK: 「進むべき方向性」「うまくいくヒント」「新しい一歩」

### 人間味を吹き込む5つの実行原則

**原則1: 構成を「会話」に崩す**
- 箇条書きや数字構成をあえて崩す
- 「ねえ、知ってます?」「ここで大事なのが…」といった自然な語りかけ
- 「間(ま)」を意図的に作る

**原則2: 感情の「起伏」と「体験」を混ぜる**
- 「最初は全然うまくいかなくて…」といった失敗談
- 「苛立ち(Frustration)」や「小さな気づき」をエピソード化

**原則3: 比喩を「日常」と「感情」に寄せる**
- NG: 「地図」「右腕」(AI的で冷たい)
- OK: 「初対面の友達に『昨日話したアレ、分かってよ』と無茶振りしてるようなもの」

**原則4: 目線を「並走者」に保つ**
- NG: 「〜しましょう!」「あなたにもできます!」(上から目線)
- OK: 「僕も悩みました」「無理しなくて大丈夫です」(並走トーン)

**原則5: 文のリズムに「呼吸」を作る**
- 短文+改行を多用
- その直後に少し長めの説明文を置く
- 文章に「緩急」と「呼吸」を生み出す

### 実行プロセス

1. **[整理]** コアな情報やロジックを整理(この時点ではAI的で可)
2. **[人間語翻訳]** 上記ルールを使って「人間味のある会話」に翻訳
3. **[最終チェック]** 「これはAIが書いたと言われても信じないか?」を確認
