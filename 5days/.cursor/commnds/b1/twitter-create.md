# B-1アウトプット作成: Twitter（500文字）

## 概要

作成済みのB-1（Belief-1）から、**Claim + Proof**の組み合わせで**Twitterコンテンツ**（500文字程度）を作成し、
{プロジェクト名}/Marketing/B1/Belief-1.mdに追記するためのガイド。

**特徴**: 
- Belief-1.mdに既に記載されているClaimとProof（Pathos/Ethos/Logos）を活用
- 過去のツイートのスタイルを学習して、あなたらしい表現で作成

## 前提条件

- ✅ B-1が作成済みであること
- ✅ Belief、Claim、Proofの方向性が決まっていること
- ✅ Proof（Pathos/Ethos/Logos）のリンク先ファイルが存在すること（または方針・説明文が記載されていること）

## Twitter作成の全体フロー

```
Step 0: プロジェクトを選択
    ↓
Step 1: Belief-1.mdを確認（Claim + Proofの一覧を抽出）
    ↓
Step 2: 過去のツイートフォルダを指定（スタイル学習）
    ↓
Step 3: Claim + Proofの組み合わせを選択
    ↓
Step 4: Proofの詳細内容を取得（リンク先ファイルまたは方針文）
    ↓
Step 5: パターンを選択
    ↓
Step 6: Tweetを作成（Claim + Proof、スタイル反映・500文字）
    ↓
Step 7: Belief-1.mdに追記
```

## Step 4: Proofの詳細内容を取得

### AIがすること

選択された**Proof**のリンク先ファイルを読み込んで、詳細内容を取得：

#### 1. リンク先ファイルが存在する場合

```

{ファイルの内容を要約}

```

**読み込み対象**:
- **Pathos**: `Story - xxx.md`または既存ファイル
- **Ethos**: `Ethos - xxx.md`または既存ファイル（研究データ、論文、実績など）
- **Logos**: `Logic - xxx.md`または既存ファイル（仕組み、ステップ、因果関係など）

**パス構造**:
- プロジェクトフォルダ: `Outputs/Course/{プロジェクト名}/Marketing/b1/`
- Wikiリンク先も同じフォルダ内を探索

#### 2. リンク先ファイルが「新規作成予定」の場合


**ユーザーへの選択肢**:
1. **方針・説明文だけで進める**（AIが内容を推測）

```

─────────────────────

【構成】
1. {フック/問いかけ/データ提示など}
2. {展開/解説/ストーリー展開など}
3. Claim: {主張部分}
4. {オチ/示唆/行動喚起など}
─────────────────────

このTweetで良いですか？
修正したい点があれば教えてください。
```

## Step 7: Belief-1.mdに追記

### AIがすること

#### 1. Belief-1.mdを読み込んで追記位置を確認

```
Belief-1.mdに以下を追記します：

## Claim: {Claim}
- Proof (Pathos - ストーリー)
    - {説明文}
        - [[Story - xxx]]
        - [[既存ファイル]]
            - [[Tweet - {タイトル}]] ← NEW（最下流・4段階目）

よろしいですか？
```

**重要**: Tweetは最下流のアウトプットなので、Story や既存ファイルよりもさらに深い**4段階目**に配置します。

#### 2. 実際に追記

`search_replace`ツールを使用して追記：

```markdown
## Claim: {Claim}
- Proof (Pathos - ストーリー)
    - {説明文}
        - [[Story - xxx]]
        - [[既存ファイル]]
            - [[Tweet - {新規}]] ← 追加（4段階目のインデント・16スペース）
```

**インデント構造**:
- 1段階目: `- Proof (Pathos - ストーリー)` （4スペース）
- 2段階目: `- {説明文}` （8スペース）
- 3段階目: `- [[Story - xxx]]` / `- [[既存ファイル]]` （12スペース）
- **4段階目: `- [[Tweet - xxx]]` （16スペース）← 最下流**

#### 3. 完了メッセージ

```
✅ 完了しました！

【更新したファイル】
- Outputs/Course/{プロジェクト名}/Marketing/b1/Belief-1.md
- Outputs/Course/{プロジェクト名}/Marketing/b1/Tweet - {タイトル}.md

Belief-1.mdに新しいTweetが追記されました。

---

次のアクション：
1. このTweetを実際に投稿する
2. 別のTweetを作成する（別のClaim + Proofの組み合わせで）
3. Mailコンテンツを作成する（/b1/mail-create を使用）
4. 他のProof（Ethos、Logos）を作成する（/b1-output-story など）

どうしますか？
```

## Tweetパターン例（500文字程度）

### 例1: Claim + Pathos（ストーリー型）

**Claim**: 「素直になれ、守破離の守が大事なんだ」
**Proof**: Pathos - 失敗体験のストーリー

```
「お前にできることは便秘の処方くらいだ」

研修医2年目の春、救急外来で指導医からそう言われた時
同期たちの前で顔から火が出るような屈辱感を覚えました。

基礎を飛ばして応用ばかり学ぼうとした結果でした。

初めはむかっとしました。
でも、素直になって、基礎から学び直すこと。

それが大事だと、指導医の教えを信じました。

今ならわかります。
これが医師人生で、最良の決断たっだと。

この時に種を守った事で、今の自分の全てがあります。
素直に、謙虚に。

一緒に学び直しましょう。
```

**構成**:
1. フック（衝撃的なセリフ）
2. ストーリー展開
3. **Claim（素直になれ、守が大事）**
4. 共感と行動喚起

---

### 例2: Claim + Ethos（データ提示型）

**Claim**: 「上流（信念）を整えるべき」
**Proof**: Ethos - Chain of Beliefの成功データ

```
5時間で1000名の募集枠が埋まった。

それが「Chain of Belief」を実装した結果です。

多くの人は、コンテンツの質や量にこだわります。
でも、それは下流の話。

本当に大切なのは、上流の設計。

顧客の信念を段階的に構築する仕組みです。

上流を整えれば、メッセージが一貫し、顧客に響く。
下流を整えても、上流がブレていれば効果は限定的。

上流から設計することで、圧倒的な成果が出ます。

あなたの発信も、上流から見直してみませんか？
```

**構成**:
1. 驚きのデータ
2. 解説（なぜそうなるのか）
3. **Claim（上流を整えるべき）**
4. 行動喚起

---

### 例3: Claim + Logos（因果関係型）

**Claim**: 「上流設計なしでは、コンテンツは響かない」
**Proof**: Logos - 因果関係の論理

```
上流設計がない
  ↓
メッセージがブレる
  ↓
顧客に響かない
  ↓
開封率が落ちる
  ↓
売上が立たない

この負の連鎖、経験ありませんか？

逆に、上流（信念）を整えれば：

一貫したメッセージ
  ↓
顧客の共感を得る
  ↓
開封率が上がる
  ↓
売上が立つ

多くの人は「何を書くか」にこだわります。
でも本当に大切なのは「なぜ書くか」。

上流を整えることから始めましょう。
```

**構成**:
1. 問題の因果関係
2. 逆の成功パターン
3. **Claim（上流設計の重要性）**
4. 行動喚起

---
